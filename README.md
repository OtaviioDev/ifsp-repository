# IFSP Academic Repository

A centralized platform developed to catalog, preserve, and showcase academic productions created by students at IFSP (Federal Institute of São Paulo). This project was originally conceived and built during the **IFSP Jacareí Hackathon**.

## 🎯 Objective

The platform addresses a real-world community challenge by centralizing institutional knowledge that is usually scattered across departments. It allows students and faculty to easily upload, organize, and discover:

* **Theses (TCCs)** and graduation papers.
* **Scientific articles** and research essays.
* **Prototypes** and innovative student projects.
* **Coursework** and shared educational resources.

## 🛠️ Tech Stack & Architecture

* **Frontend:** Semantic HTML5, CSS3, and Vanilla JavaScript (ES6+).
* **Build Tool:** Vite for modern development workflow and optimized builds.
* **Backend Services:** Supabase for database integration and cloud services.
* **Authentication:** User authentication system for students and professors.
* **Architecture:** Modular JavaScript structure with separated authentication, services, and interface logic.

## 🚀 Current Features

* **Dynamic Search Engine:** Real-time project filtering through keyword searches.
* **Course-Based Indexing:** Organization of projects by academic fields and courses.
* **User Authentication:** Login system with different user profiles.
* **Supabase Integration:** Connection with cloud database services.
* **Responsive Interface:** Adapted layout for different screen sizes.
* **Project Management:** Structure prepared for academic document organization and expansion.

## 💻 Hackathon Development & AI Integration

This functional MVP was developed as a collective team effort during the **IFSP Jacareí Hackathon**, under a high-pressure and time-constrained environment.

### Development Process

To accelerate research, prototype new features, and improve development speed, our team integrated Artificial Intelligence tools, including **DeepSeek**, during the development process.

### Quality Assurance

AI-generated suggestions and code snippets were reviewed, adapted, tested, and integrated manually by the team to ensure code quality, maintainability, and architectural control.

## 📈 Next Milestones

1. Expand document storage and management features.
2. Improve authentication and permission control between students and professors.
3. Develop a complete academic catalog system.
4. Improve accessibility and user experience.
5. Deploy the platform for public access.

# IFSP Repository

Uma plataforma centralizada desenvolvida para catalogar, preservar e divulgar produções acadêmicas criadas pelos estudantes do IFSP (Instituto Federal de São Paulo). O projeto foi idealizado e desenvolvido originalmente durante o **Hackathon do IFSP Jacareí**.

## 🎯 Objetivo

A plataforma busca resolver um problema real da comunidade acadêmica: a dispersão de conhecimentos e trabalhos produzidos pelos estudantes em diferentes locais.

O sistema permite organizar e futuramente disponibilizar:

* **TCCs** e trabalhos de conclusão de curso.
* **Artigos científicos** e pesquisas acadêmicas.
* **Protótipos** e projetos inovadores.
* **Trabalhos acadêmicos** e materiais educacionais.

## 🛠️ Tecnologias Utilizadas

* **Frontend:** HTML5, CSS3 e JavaScript.
* **Build Tool:** Vite.
* **Banco de Dados e Serviços Cloud:** Supabase.
* **Autenticação:** Sistema de login para estudantes e professores.

## 🚀 Funcionalidades Atuais

* Busca dinâmica de projetos.
* Organização por curso e área acadêmica.
* Sistema de autenticação de usuários.
* Integração com banco de dados através do Supabase.
* Interface responsiva.

## 💻 Desenvolvimento no Hackathon e Uso de IA

O projeto foi desenvolvido em equipe durante o Hackathon do IFSP Jacareí.

Durante o desenvolvimento, foram utilizadas ferramentas de Inteligência Artificial, incluindo o **DeepSeek**, para auxiliar na pesquisa, geração de código e aceleração do desenvolvimento.

Todo código gerado ou sugerido por IA foi revisado, adaptado e testado pela equipe, garantindo controle sobre a arquitetura e funcionamento da aplicação.

## 📈 Próximos Passos

1. Expandir o sistema de armazenamento e gerenciamento de documentos.
2. Melhorar os níveis de permissão entre estudantes e professores.
3. Desenvolver um catálogo acadêmico mais completo.
4. Aprimorar acessibilidade e experiência do usuário.
5. Realizar o deploy da plataforma.
